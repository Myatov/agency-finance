# Внутренняя инструкция по формированию счёта на оплату (PDF)

*Этот документ можно править. По нему заполняется отчёт при нажатии «Печать отчета».*

## Общие правила (Россия)

- Счёт на оплату не имеет унифицированной формы; структура — по договорённости и обычаю.
- В документе должны быть однозначно определены: кто платит, кому платят, за что, сумма, реквизиты для перевода.

## Структура счёта в нашем отчёте (PDF)

1. **Заголовок**  
   Счёт № [номер]. Номер берётся из поля «№ счета» на странице (можно править только для печати).

2. **Реквизиты плательщика (клиент)**  
   Блок заполняется из карточки Клиента: наименование, адрес, ИНН/КПП, ОГРН, р/с, банк, БИК/КС, прочие платёжные реквизиты, контакты. Пустые поля не выводятся.

3. **Реквизиты получателя (юрлицо)**  
   Блок заполняется из справочника Юрлица (юрлицо, выбранное по счёту): наименование, адрес, ИНН/КПП, ОГРН, р/с, банк, БИК/КС, платёжные реквизиты, гендиректор, основание деятельности. Пустые поля не выводятся.

4. **Содержание**  
   - Сайт — из периода (сайт услуги).  
   - Услуга — из поля «Услуга (для печати)» на странице (можно править только для печати).  
   - Период — в формате ДД.ММ.ГГГГ — ДД.ММ.ГГГГ.  
   - Сумма (руб) — сумма счёта.

5. **НДС**  
   Если у юрлица (получателя) в справочнике указан НДС % и он больше 0:  
   - НДС (руб) = Сумма × (НДС % / 100).  
   - Сумма с НДС (руб) = Сумма + НДС (руб).  
   Эти строки добавляются в отчёт только при ненулевом НДС.

6. **Имя файла при скачивании**  
   `№ счета.pdf` (например, `INV-20260207-abc123.pdf` или введённый пользователем номер).

## Что можно править только для печати (в БД не сохраняется)

- Номер счёта (№ счета).  
- Наименование услуги (Услуга).
